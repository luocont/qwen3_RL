# AI病例分析工作流 - 项目总览

## 项目简介

这是一个基于5Ps个案概念化框架和理情行为疗法（REBT）的AI心理咨询工作流系统。系统通过两个专业模型协作，为患者提供结构化的病例分析和专业的心理咨询服务。

## 核心文件说明

### 1. 主要程序文件

| 文件名 | 功能 | 使用场景 |
|--------|------|----------|
| `run.py` | 快速启动器 | 新手推荐，提供菜单式交互 |
| `main.py` | 命令行入口 | 支持命令行参数，适合脚本调用 |
| `workflow.py` | 工作流核心 | 包含所有业务逻辑 |
| `config.py` | 配置文件 | 管理API密钥和路径配置 |
| `test_workflow.py` | 测试脚本 | 功能验证和调试 |

### 2. 提示词文件

| 文件名 | 功能 |
|--------|------|
| `prompts/five_ps_teacher.py` | 5Ps病例分析模型提示词 |
| `prompts/think_and_reply_teacher.py` | 思考与回复模型提示词 |

### 3. 数据文件

| 文件名 | 说明 |
|--------|------|
| `RL.json` | 输入数据（患者对话记录） |
| `workflow_results.json` | 输出结果（病例分析和对话历史） |

## 快速开始

### 第一步：安装依赖
```bash
pip install -r requirements.txt
```

### 第二步：配置API密钥

**方法A：环境变量（推荐）**
```bash
export DASHSCOPE_API_KEY="your-api-key"
```

**方法B：配置文件**
编辑 `config.py`：
```python
DASHSCOPE_API_KEY = "sk-your-actual-api-key"
```

### 第三步：运行程序

**方式1：使用快速启动器（推荐新手）**
```bash
python run.py
```

**方式2：命令行模式**
```bash
# 批处理模式
python main.py --mode batch --max-rounds 3

# 交互式模式
python main.py --mode interactive

# 单次处理
python main.py --mode single --message "我最近感到很焦虑"
```

## 输出格式示例

### 5Ps病例输出
```xml
<5Ps>
P1 主诉：睡眠障碍、情绪低落、日常功能受损

P2 易感因素：待完善

P3 诱发因素：工作压力、上级要求过高

P4 维持因素：反刍思考、睡眠紊乱

P5 保护因素：待完善
</5Ps>
```

### 思考与回复输出
```
思考阶段:前期

当前患者主要表现为焦虑和睡眠问题，5Ps病例尚不完善。需要进一步了解患者的易感因素、诱发事件的具体细节，以及可能的保护因素。

我理解你现在的感受很不容易。睡眠问题确实会影响我们的情绪状态。你提到工作压力是主要原因，能具体说说是什么样的工作压力吗？
```

## 技术架构

### 模型配置
- **病例分析模型**: Qwen-Max
- **思考回复模型**: Qwen3-Max（深度思考）

### API集成
- 使用阿里云百炼（DashScope）API
- 兼容OpenAI API格式
- 支持流式输出

### 数据处理
- 自动提取JSON数据中的用户输入
- 支持多轮对话历史管理
- 每个会话独立的历史记录

## 常见问题

### Q1: API密钥在哪里获取？
A: 需要前往阿里云百炼平台申请API密钥。

### Q2: 如何处理大量数据？
A: 使用批处理模式，可以通过`--max-rounds`参数控制处理轮数。

### Q3: 历史记录如何管理？
A: 系统自动维护5Ps病例和对话历史，每次对话都会更新，每个会话独立。

### Q4: 如何自定义提示词？
A: 可以编辑`prompts/`目录下的提示词文件，但需保持变量占位符格式。
